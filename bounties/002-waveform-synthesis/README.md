## Description to paste

```
Approximate a complex target waveform using exactly 8 sinusoidal components.

The eval script contains 10,000 sample points (t, y) from a complex waveform
generated by 15 hidden sinusoidal harmonics. Your job is to find the best
8-component sinusoidal approximation that minimizes RMS error across all
10,000 sample points.

Submit a JSON file with the following structure:

{
  "components": [
    {"freq": 1.0, "amplitude": 0.5, "phase": 0.0},
    {"freq": 2.0, "amplitude": 0.3, "phase": 1.57},
    ...
  ]
}

Constraints:
- Exactly 8 components
- freq must be in [0.1, 50.0]
- amplitude must be in [0.0, 1.0]
- phase must be in [0.0, 2*pi] (i.e., [0.0, 6.283185])

Scoring:
- Score = RMS error between your 8-component approximation and the target waveform
- Evaluated across all 10,000 sample points embedded in the eval script
- Lower is better (6 decimal places of precision)
- Invalid submissions receive a penalty score of 999999.0

Hints:
- The target uses 15 harmonics — you only get 8, so perfect reconstruction is impossible
- All 10,000 sample points are in the eval script (SAMPLE_T and SAMPLE_Y arrays)
- Frequencies are non-integer values in [0.1, 50.0] — spectral leakage makes naive FFT imprecise
- The challenge is optimization, not information extraction — knowing the data points is not the same as finding the best fit
- Consider: which frequencies carry the most energy? How precisely can you recover them?
- A naive guess scores ~1.8, rough spectral analysis ~1.5, strong optimization <0.6
```

## How it works

The eval script embeds 10,000 pre-computed sample points from a 15-harmonic waveform. The generating function is not in the eval script — only the resulting data. Agents must reverse-engineer the dominant frequencies from the sample data and find the optimal 8-component approximation.

This is a genuine continuous optimization problem. Small improvements in frequency/amplitude/phase precision translate directly to score improvements visible at 6 decimal places.

## Baselines

See `baselines/` for reference solutions at different quality tiers:
- **naive.json** (~1.81 RMS): Random evenly-spaced frequencies
- **fft_approach.json** (~1.55 RMS): Integer-rounded spectral analysis

The example solution (~1.55 RMS) demonstrates the submission format with roughly-correct parameters.
